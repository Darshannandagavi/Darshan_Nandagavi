Jarvis is a local personal assistant / automation project that combines voice interaction, simple web UI, modular skill handlers and system helpers. The repository groups core memory/command parsing, modular capabilities (email, music, social, system), and a small frontend for text-based interactions and history viewing.

1. Project Summary

Jarvis is designed as a lightweight assistant that can listen (speech recognition), respond (text + TTS), perform tasks via modular handlers (play music, send email, control system), and persist conversational history. The codebase separates responsibilities into a modules/ folder for actionable skills, a core/ folder for parsing, memory and task orchestration, and a frontend/ for a minimal web UI to send commands and view history.

2. Tech Stack

Python 3 (core logic & modules)
Browser speech recognition & Web Speech API (frontend)
smtplib / email.mime (email helper)
JSON based chat history (core/chat_history.json)
OS-level commands and system control helpers
Vanilla JS frontend (frontend/index.html)

3. High-level Control Flow

  1. User provides input via voice or the web UI â†’ frontend posts to the assistant endpoint (e.g., POST /ask).
  2. Core parser (command_parser.py / task_handler.py) interprets the command and extracts intent and parameters.
  3. Dispatcher routes the request to an appropriate module (modules/email_sender.py, modules/music_player.py, modules/system_control.py, etc.).
  4. Module executes the action (send email, play music, control system) and returns a textual response.
  5. The assistant saves the exchange to core/chat_history.json and returns the response to the frontend; the frontend may render + speak the response via Web Speech API.

Key Components

  • modules/ - Skill modules (email_sender.py, music_player.py, voice.py, social_media.py, system_control.py, etc.).
  • core/ - Command parsing, memory, task orchestration and chat history handling.
  • frontend/ - Minimal web UI (index.html) used to send commands and view history.
  • Environment - Uses environment variables (for example EMAIL_ADDRESS and EMAIL_PASSWORD) for credentials and sensitive configuration.

Implementation Notes

Command parsing: modular parser that normalizes input and maps it to handler functions.

Email helper: builds MIME messages and sends via smtplib; keep credentials in env vars.

Voice: frontend uses Web Speech API for recognition and SpeechSynthesis for TTS; modules.voice may offer local TTS as well.

6. Features

Built-in Features

  • Voice input and continuous listening (frontend uses Web Speech API).
  • Text responses + TTS playback via browser SpeechSynthesis.
  • Modular skill handlers: send email, play music, social actions, system commands.
  • Persistent chat history stored as JSON (core/chat_history.json).
  • Simple contact lookup table for quick email addresses (modules/email_sender.py CONTACTS).

Future Enhancements

  • Integrate a small HTTP API (Flask/FastAPI) for the Python backend to serve the frontend cleanly.
  • Improve natural language parsing and intent recognition (use lightweight ML/NLP or regex rules).
  • Secure credential storage and add retry/failure handling for networked modules (email).
  • Unit tests for core parsers and modules; CI to run static checks.
  • Optional CI/CD packaging to run Jarvis as a background service on the workstation.

7. Development & Run Instructions

Local Setup (Python)

# Create a virtual env and install deps (if any listed in requirements.txt)
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt  # if present

# Set required environment variables (Windows PowerShell example)
$env:EMAIL_ADDRESS="you@example.com"
$env:EMAIL_PASSWORD="your-email-password"

# Run the main assistant process (adjust entrypoint if different)
python main.py   # or python app.py / python server.py depending on repo

Frontend

# Serve frontend/index.html statically (open directly or run a simple server)
# From the frontend folder you can run:
py -m http.server 8000
# Then open http://localhost:8000 in the browser

Ensure environment variables for credentials are set and that any optional Python dependencies (speech, TTS libraries) are installed. Check modules/email_sender.py for CONTACTS and required email settings.
