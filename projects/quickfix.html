A full-stack service-booking platform with separate roles (guest, user, shop owner, admin). React front-end (client/) and Node/Express backend (server/). Core features: service listings, booking flow, file uploads for shop images, user/shop owner management and admin controls.

1. Project Summary

Quick-Fix is a marketplace-style application where users can browse shop services, book appointments, and communicate with shop owners. The repository contains a React client (client/) and an Express-based server (server/) with controllers, routes, middleware and models for Users, Bookings, Services and Notifications. File upload handling and email utilities are included on the server side.

3. High-level Control Flow

  1. User opens the React app â†’ App mounts via client/src/index.js â†’ main UI components under client/src/components handle routes and state.
  2. Guest browses services â†’ select a service and open booking page (client-side BookPage/BookPage components).
  3. User submits a booking â†’ client POSTs booking data to server (bookings endpoint) â†’ server validates, creates Booking model entry and notifies shop owner.
  4. Shop owner views bookings via ShopOwner dashboard â†’ accept/decline or update status â†’ server updates Booking and optionally sends email/notification.
  5. Admin functions allow approving shop owners, managing users and viewing all bookings and content via admin controllers and routes.

Key Backend Components

  • server/index.js - Express entry point, middleware and route mounting
  • Controllers - AdminController, UserController, ContactController, shopServiceController manage business logic
  • Routes - routes/ exposes REST endpoints for users, services, bookings and admin actions
  • Multer & upload helpers - config/bookingmulter.js and multerConfig.js handle file uploads for propertyImages/uploads folders

Client Structure

client/src/App.js: Main application routes and layout selection (guest, user, shop owner, admin)

client/src/components/ - UI components organized by role: guestLayout, userLayout, shopOwnerLayout, adminLayout

client/src/context/AuthContext.js: Authentication state and helpers used across the client

7. Development Setup

Server Setup

Client Setup

cd client

npm start

The server contains upload folders (uploads/, propertyImages/) and helper config for Multer. Ensure environment variables in server/.env are set (JWT keys, DB connection string, mail config) before starting.
