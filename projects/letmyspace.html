LetMySpace — Project Overview

Full-stack property listing and booking platform

1. Project Summary

LetMySpace is a full‑stack web application for listing and discovering rental/sale properties. The project has a Node/Express server with MongoDB models and email/file upload logic, and a React front-end with layouts for guest, user, and admin.

2. Tech Stack

React
Node.js
MongoDB
Express.js
Nodemailer
Multer

3. High-level Control Flow

  1. User registers → server validates → generates verification code → email sent → user confirms code → account saved
  2. User logs in → server verifies credentials → updates userStatus → returns basic user object
  3. User uploads property → images uploaded via /propertyImages endpoint (Multer) → image paths saved with property record → confirmation email sent to owner
  4. Client fetches properties from /user/properties to display listings (UserHome component)
  5. Admins and other routes operate via dedicated routers and controllers under server/controller and server/routes

Key Backend Components

  • index.js - Server entry point with MongoDB connection
  • UserController - Handles registration, login, verification
  • Property handling - Image uploads and property CRUD
  • Email system - Verification and notifications

Database Models

User Model: firstName, lastName, email, password, role, userStatus, profilePicture, contact info

PropertyData Model: propertyType, owner, size, location, images, description, amenities

4. System Architecture

(Architecture flow diagram placeholder)

Frontend Structure

  • Guest Layout - Public pages (Home, About, Register, Login)
  • User Layout - Authenticated pages (UserHome, PropertyUploader, MyProperties)
  • Admin Layout - Admin dashboards and user management

Data Flow Examples

  • UserHome fetchProperties(): axios GET to /user/properties, stores results in state and renders list
  • PropertyUploader: uploads image(s) to /propertyImages, receives filepath(s), then posts property metadata
  • Forgot password: client requests /user/forgot/:email → backend generates temp password, emails it and updates DB

Component Interconnection

Browser (React) --axios--> Express routes (/user, /admin, /category, ...)
Express route --> Controller action --> Model (Mongoose) --> MongoDB
Controller may call: Nodemailer (send email), Multer (save files), filesystem (serve static)

5. API Endpoints

POST /user/register POST /user/login POST /user/email-verification GET /user/properties POST /propertyImages POST /upload GET /uploads/:filename
